Chapter 12 - Object Oriented Programming

OOP has 3 main concepts:

  1. Encapsulation - inner workings are hidden and only essential functions are available
  2. Polymorphism - various objects can use the same method
  3. Inheritance - take features of one object and add new ones

Prototype-based = used by javascript before classes became available.

ES6 class declaration works just like a constructor function. Name of class is capitalized (e.g. class Dice {})

Static Methods - (called class methods elsewhere) called by the class directly rather than instances of the class. (e.g. Dice.description() but not redDice.description, redDice being an instance of Dice).

Prototypal inheritance - every class has a prototype property that is shared by every instance of the class. Can you change a prototype property into an own property? Yes, assign them their own value. These will take precendence over prototype props. Prototype cannot be overwritten by assigning it to a new object literal if class declarations are used. Use constructor functions to overwrite. However, any instances already existing will keep the old properties.

Prototypes are used to add properties or methods after a class has been declared. Do not use arrays or objects for a default value.

Use private properties to limit access to properties. Changes can be validated this way as well.

Properties that are non-enumerable will not show up in a for-in loop.

All properties and methods created by assignment are enumerable.

Built-in methods should be non-enumerable, user-defined methods should be enumerable.

The extends keyword in a class declaration let an object inherit class from another class (class NinjaTurtle extends Turtle)

Polymorphism. toString() method is shared by all objects. Polymorphism means than an object can override this with a more specific implementation
