  • Soft equality (==) - does not take type into account when comparing values and may force values
  • Hard equality (===) - will look at type when comparing values. Except NaN === NaN evaluates as false. So you have to use Number.isNaN(NaN) to get 'true'. This answers my question above from p82. :) Always use === while keeping in mind the NaN anomaly and explicitly convert types before comparing.
  • Inequality - similarly there is soft inequality (!=) and hard inequality (!==). The former suffers from type coercion, the latter is the preferred method.
