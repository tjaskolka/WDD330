Week 5 Notes and Code

Errors are usually caused by one of the following:

  • System error
  • Programmer error
  • User error

It is the programmers responsibility to be aware of possible system errors and work around them, to fix errors in programming ASAP and to anticipate possible user errors and provide ways to deal with them that will help the user.

EXCEPTION: error that produces a return value that can be used by the program to deal with error.

STACK TRACES: sequence of functions or method calls that lead to the point where the error occurred.

WARNINGS: an error that isn't enough of a problem to stop the program from functioning.

STRICT MODE: a less forgiving way of running the program that will catch and identify more exceptions and warnings and especially sloppy code practices. To use strict mode, just add the following as the first line of the javascript file: 'use strict';. Strict mode can also be used inside a function to limit it to that function. It is recommended that you place all your code in a function that will invoke strict mode:

    (function() {
        'use strict';
        //All your code would go inside this function
    }());

Javascript modules are strict by default and do not require the 'use strict' declaration.

Linting tools - used to catch syntax errors and can enforce proper style and conventions.

It is recommended to check for browser support of desired features before calling the method, preventing exceptions that might cause issues. (see Modernizr or Can I Use?)

Use alert() when debugging to check expected value(s) at certain point, as well as stop the process at that point (until you click 'OK').

Use console.trace() to show chain of functions called up to that point.

Use debugger command within code to pause execution and check variables.

Errors:

  • EvalError - no longer used
  • RangeError - number is outside allowable range
  • ReferenceError - reference to item that doesn't exist
  • SyntaxError - problem with code syntax
  • TypeError - unexpected type of value being used
  • URIError - problem with the URI
  • InternalError - error with Javascript engine (e.g. recursion)

Error properties (to be used for custom errors):

  • name - returns name of (custom error) function
  • message - description of error
  • stack - stack trace of error (not for production use)

Use 'throw' to create your own exceptions (not covered by that above)

try - used to wrap a suspect piece of code that will pass the error to a 'catch' block, where you can diagnose and deal with the error.

finally - code that will run after try and catch regardless of which ran before.

TDD - Test Driven Development - writing tests before you write the code. Then write code to pass the tests, then streamline the code while testing all along. Full process:

  1. Write tests
  2. Write code to pass tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features

Expectation object - returned by test function

matcher - information provided to test to check function output (if function works correctly, they should match)
